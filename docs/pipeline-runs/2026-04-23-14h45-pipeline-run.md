### Bug #2 — TRAITÉ (commits 20b0395 + 0024226)

**Cause racine multi-couches** identifiée via inspection du NFO `Top Chef Le Concours Parallèle (2026)/tvshow.nfo` :
1. `<uniqueid default="true" type="tmdb">0</uniqueid>` — corrigé en amont par `a53a44f` (avant ce run).
2. `<thumb>https://image.tmdb.org/t/p/originalhttps://artworks.thetvdb.com/banners/v4/series/475278/posters/69b.jpg</thumb>` — **non corrigé jusqu'ici** : `_add_inline_images_tv` (+ `_add_inline_images` + `still_path` episode) préfixait `IMAGE_BASE/original` sans vérifier si `file_path` était déjà une URL absolue. TVDB (fallback quand TMDB absent) renvoie des URLs complètes.
3. NFO jamais régénéré car `is_nfo_complete` acceptait `<uniqueid>0</uniqueid>`. Quiconque avait scrapé un show TVDB-only avant `a53a44f` restait avec un NFO cassé indéfiniment.

**Fix** :
- `nfo_generator.py` : helper `_image_url(path, size)` — retourne `path` tel quel s'il commence par `http://`/`https://`, sinon préfixe. Toutes les 3 sites (movie `_add_inline_images`, TV `_add_inline_images_tv`, episode still) passent par lui. 4 tests ajoutés.
- `nfo_utils.py` : `is_nfo_complete` rejette `{"0", "none"}` (case-insensitive). Si un NFO a plusieurs uniqueids dont au moins un vrai, il reste valide. 3 tests ajoutés.

**Suite** : le prochain run `personalscraper process` régénèrera automatiquement les NFO de tous les shows TVDB-only scrapés avant `a53a44f` (ils redeviennent "incomplete" donc re-scrape).

### Bug #3 — TRAITÉ (commit 3231b8d)

**Cause racine** : `_normalize_key` dans `personalscraper/dispatch/media_index.py` faisait seulement `name.lower().strip()`. Sur un système où staging (APFS/HFS, NFC) et destination (NTFS-via-macFUSE, NFD) coexistent, le même titre produit deux clés distinctes après merge.

**Fix** : `unicodedata.normalize("NFC", name).lower().strip()`. 2 tests de régression ajoutés (lookup NFC↔NFD, dé-duplication à l'`add`).

**Effet sur index existant** : les entrées NFD legacy restent trouvables via le fuzzy-match (`text_utils.fuzzy_match_score` strippe déjà les combining marks). Le prochain `add()` d'un item NFC collapsera proprement.
