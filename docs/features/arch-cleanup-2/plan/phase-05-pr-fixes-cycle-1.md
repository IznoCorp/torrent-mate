## Sub-phase 5.1 — CRITICAL: registry events round-trippable (NameError fix)

**Finding:** `event_from_envelope` raises `NameError` for `ProviderExhaustedEvent`,
`RegistryFanOutCompleted` (`tuple[AttemptOutcome, ...]`) and `LockedCapabilityUnresolved`
(`match: ProviderMatch`) because `_events.py` has `from __future__ import annotations` and
imports `AttemptOutcome`/`ProviderMatch` only under `if TYPE_CHECKING:`; `get_type_hints`
can't resolve them at runtime. This breaks the Phase-1 catalog round-trip guarantee.

**Fix (approach a — break the import cycle via a leaf module):**

- `AttemptOutcome` and `ProviderMatch` currently live in `api/metadata/registry/__init__.py`,
  which imports `_events`. A runtime import of them into `_events.py` would cycle.
- Create `personalscraper/api/metadata/registry/_types.py` (leaf — must NOT import from
  `registry/__init__.py`) and MOVE `AttemptOutcome` + `ProviderMatch` (and any small type
  they require, e.g. `RegistryProviderName`, that would otherwise re-introduce the cycle) there.
- `registry/__init__.py`: re-export them from `_types.py` (explicit `X as X`, mypy strict) so
  the public path `from personalscraper.api.metadata.registry import AttemptOutcome, ProviderMatch`
  still resolves for all callers.
- `_events.py`: import `AttemptOutcome`/`ProviderMatch` from `_types.py` at **runtime** (drop
  the `TYPE_CHECKING` guard for them). Keep `from __future__ import annotations`.
- If `_types.py` cannot be made cycle-free, fallback: inject the names into `_events`' module
  globals at registry import time, OR `event_from_envelope`/`get_type_hints` resolve against a
  namespace including them. Prefer the leaf-module approach.

**Regression test (mandatory — per project rule):** add a test that round-trips EVERY
registered event (drive off `_EVENT_CLASS_REGISTRY` + `EVENT_SAMPLE_FACTORIES`) through
`event_from_envelope(json.loads(json.dumps(event_to_envelope(ev))))` and asserts `== ev`.
This must FAIL on the current code and PASS after the fix.

**Acceptance:**

```bash
python -c "
import json, personalscraper.events
from personalscraper.core.event_bus import _EVENT_CLASS_REGISTRY, event_to_envelope, event_from_envelope
from personalscraper.tests_fixtures import *  # if needed
" 2>/dev/null || true
python -m pytest tests/architecture/test_registry_events_contract.py tests/event_bus/ -q   # all pass
python -m mypy personalscraper/   # 0 errors (no import cycle)
python -c "from personalscraper.api.metadata.registry import AttemptOutcome, ProviderMatch; print('public path ok')"
```

## Sub-phase 5.3 — MINOR code: fail-loud heterogeneous-tuple decode

**Finding:** `_decode_field_value` heterogeneous-tuple branch (`event_bus.py` ~line 79-82) uses
`zip(value, args)` — silently truncates on length mismatch. Latent (no event hits it yet).

**Fix:** use `zip(value, args, strict=True)` (or an explicit length check that raises
`ValueError`) so a malformed envelope fails loud. Add a unit test: decoding a wrong-length
value against `tuple[X, Y]` raises (not silently truncates).
