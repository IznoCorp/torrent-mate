### 2.1 — Password hashing + JWT tokens

**Commit**: `feat(tm-shell): add password hashing and JWT token helpers`

**Files**:

| Action | Path                                    |
| ------ | --------------------------------------- |
| Create | `personalscraper/web/auth/__init__.py`  |
| Create | `personalscraper/web/auth/passwords.py` |
| Create | `personalscraper/web/auth/tokens.py`    |

**Work**:

1. `auth/passwords.py` — `hash_password(password: str) → str` using
   `hashlib.scrypt`, format `scrypt$N$r$p$salt_b64$hash_b64` (DESIGN §4.1).
   `verify_password(password, stored) → bool` with constant-time compare.
2. `auth/tokens.py` — `create_session_token(username, secret, ttl_hours) → str`
   (PyJWT HS256, `{sub, iat, exp}`). `decode_session_token(token, secret) →
dict | None` (catches ExpiredSignatureError → None).
3. Pure functions, no FastAPI/request dependency — testable standalone.

**Verification**: standalone doctest or quick `python -c` round-trip:
`verify_password("pw", hash_password("pw"))` → `True`.

### 2.2 — Auth routes + guard dependency

**Commit**: `feat(tm-shell): add login/logout/me routes and require_session guard`

**Files**:

| Action | Path                                 |
| ------ | ------------------------------------ |
| Create | `personalscraper/web/auth/routes.py` |
| Modify | `personalscraper/web/deps.py`        |
| Modify | `personalscraper/web/app.py`         |

**Work**:

1. `deps.py` — add `require_session` FastAPI dependency: reads `tm_session`
   cookie, decodes JWT, raises 401. Single-user: extracts config `username`,
   verifies against token `sub`. Returns `Session( username=...)` pydantic model.
2. `auth/routes.py` — mount `APIRouter(prefix="/api/auth")`:
   - `POST /login` — `{username, password}` body → verify scrypt hash → JWT →
     Set-Cookie `tm_session` (HttpOnly, SameSite=Strict, Secure if configured,
     Path=/). 204 on success, 401 + small `time.sleep(0.1)` on failure.
   - `POST /logout` — clear cookie, 204.
   - `GET /me` — depends on `require_session`, returns `{username}`.
3. `app.py` — mount auth router; apply `require_session` guard to existing
   health/version routes (or keep health/version public → operator choice:
   DESIGN says guard on every `/api/*` except login, so health stays public
   as a liveness probe but version gets guarded).

**Verification**: `python -c "from personalscraper.web.auth.routes import router"` imports.
