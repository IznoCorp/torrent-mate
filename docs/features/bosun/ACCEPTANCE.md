## Notes

- **CI scope.** CI exercises every route + job + validator with the GitHub/PM2 boundaries stubbed
  (monkeypatched `ops.create_job`, fake seeders, fake `subprocess`). It cannot bounce real PM2 apps
  or mutate a real GitHub board, so ACC-03/05/07/09/12's _outward_ effects carry a post-merge
  live-verify step (DESIGN §17). The route + job _wiring_ IS proven in CI.
- **Security model.** CSRF double-submit is enforced only when auth is enabled (a session exists to
  protect); `/api/login` + `/api/logout` are exempt; the `km_csrf` cookie is always minted (DESIGN §6,
  refined at the 2.3 gate). Privileged dashboard data rides the authed `/api/admin/*` surface, never
  the public `/api/health`.
- **Every automated proof above passes** under `make check` on this branch (3.12 / `[dev,ui,mcp]`).
