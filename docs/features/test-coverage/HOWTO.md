# Adding a design-contract test (3-step quick-start)

A design-contract test pins one observable behavior described in a `DESIGN.md` (or reference doc) section. The test docstring carries `Design:` and `Contract:` markers that are scanned by `scripts/update_feature_map.py` and audited by `scripts/audit_design_coverage.py` so every documented behavior is traceable to a passing test.

## 1. Identify the anchor

Pick the section heading you want to validate. Compute the GitHub-style anchor (or read it from the audit output):

```bash
python3 scripts/audit_design_coverage.py | grep -i "<keyword>"
```

The output is `error|warn: [orphan-section] <design-path>: section '#<anchor>' has no tests`. Copy the `<design-path>` and `<anchor>` verbatim.

If you'd rather compute the anchor yourself:

```bash
python3 -c "import sys; sys.path.insert(0, 'scripts'); \
  from audit_design_coverage import github_anchor; \
  print(github_anchor('Your Heading Text'))"
```

## 2. Write the test

Create or extend `tests/integration/test_design_<module>.py`. The test docstring must carry **both** markers:

```python
def test_my_contract(staging_dir):
    """Short behavioral description.

    Design: docs/features/<feature>/DESIGN.md#<anchor>
    Contract: One sentence describing the observable behavior the test
    pins down.
    """
    ...
```

Conventions:

- **One test = one contract clause.** If the section describes three rules, write three tests.
- **Behavioral assertions only** — assert observable outcomes, never internal call patterns.
- **Both markers required** — `Design:` without `Contract:` is skipped with a warning by `update_feature_map.py`.
- **Cross-cutting tests** may carry multiple `Design:` lines pointing at different anchors (or different docs).

## 3. Commit

```bash
git add tests/integration/test_design_<module>.py
git commit -m "test(<scope>): contract test for <behavior>"
```

The pre-commit hook (`hooks/pre-commit`, installed by `hooks/install.sh`) regenerates `tests/feature_map/<codename>.json` and stages it as part of the same commit. CI's `design-gaps` job verifies the map is in sync via `update_feature_map.py --check` and runs `audit_design_coverage.py` so stale or missing references trip a build before a reviewer has to.

## Troubleshooting

| Symptom                                                  | Cause                                                                        | Fix                                                                                                                                                                         |
| -------------------------------------------------------- | ---------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Hook prints `feature maps up to date` but no map appears | Test docstring missing the `Contract:` marker                                | Add a `Contract:` line; the script skips Design-only docstrings with a warning.                                                                                             |
| `update_feature_map.py --check` fails in CI              | Map file committed without running the hook (or `--no-verify`)               | Re-run `python3 scripts/update_feature_map.py` locally and amend.                                                                                                           |
| `audit_design_coverage.py` reports a stale reference     | The DESIGN heading was renamed but the marker still points at the old anchor | Update the `Design:` line to match the new anchor; map is regenerated by the hook.                                                                                          |
| Anchor mismatch only on macOS                            | NFD bytes from a Finder paste-as-text                                        | The audit normalizes to NFC, so equality holds — but if you wrote the marker yourself with NFD characters, retype the anchor in NFC (most editors normalize automatically). |

## 6-month maintenance audit

`scripts/coverage_audit_report.sh` produces a dated report at `docs/features/test-coverage/audit-YYYY-MM-DD.md` capturing branch coverage, design coverage with `--strict-skip` (so expired `skip_audit` entries surface), and map freshness. Schedule it twice a year via cron:

```cron
# 1st January and 1st July at 09:00 — coverage audit
0 9 1 1,7 * cd $HOME/dev/PersonnalScaper && ./scripts/coverage_audit_report.sh
```

On macOS prefer launchd; the personalscraper launchd helpers in `scripts/install-launchd.sh` are the existing reference. The audit output is committed manually only if it surfaces actionable findings (expired waivers, coverage regression, new orphan sections).
