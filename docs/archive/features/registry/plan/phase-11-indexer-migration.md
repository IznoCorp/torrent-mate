# Phase 11 — Indexer migration (delayed-by-design)

Generated by the post-merge retrospective. Addresses item #18 of the 30-point audit. The only real consumer of `RatingProvider.get_notations()` (`indexer/backfill_ids.py`) was deliberately kept out-of-scope per DESIGN §11 ("logical pairing with the P1 Library/Indexer Consolidation roadmap item"). This phase delivers the missing consumer migration.

## Gate

- Phases 7+8+9 complete (registry framework hardened, chain semantics in production).
- `indexer/backfill_ids.py` currently takes typed `TMDBClient | None` / `TVDBClient | None` / `IMDbClient | None` / `RottenTomatoesClient | None` parameters.
- ProvidersConfig.RatingProvider has providers configured (e.g. `tmdb: 1, omdb: 2, rotten_tomatoes: 3, trakt: 4`).

## Goal

Migrate `personalscraper/indexer/backfill_ids.py` (and its callers in `commands/library/scan.py`) from direct typed clients to `registry.fan_out(RatingProvider)` for ratings aggregation. Migrate `_resolve_tmdb_id` family functions to use `registry.chain(MovieDetailsProvider | TvDetailsProvider)` where applicable.

## Scope

- `personalscraper/indexer/backfill_ids.py` — the main file.
- `personalscraper/indexer/scanner/_modes/backfill_ids.py` — the indexer scanner mode that calls into it.
- `personalscraper/commands/library/scan.py` — `library_backfill_ids` Typer command, currently extracts typed clients via `registry.get(name)` and passes them down.
- Tests in `tests/indexer/` covering backfill_ids.
- `tests/feature_map/indexer.json` — re-mapped to registry-aware tests.

## Sub-phases

### 11.1 — Audit `indexer/backfill_ids.py` provider usage

Inventory every site where `tmdb_client`, `tvdb_client`, `imdb_client`, `rt_client` are USED inside `backfill_ids.py`:

```bash
rg "tmdb_client\.|tvdb_client\.|imdb_client\.|rt_client\." personalscraper/indexer/ --type py
```

Classify each call:

- **Ratings aggregation** → migrate to `registry.fan_out(RatingProvider)`.
- **ID validation / cross-ref** → migrate to `registry.get(name)` (direct dispatch).
- **Details fetch** → migrate to `registry.chain(MovieDetailsProvider | TvDetailsProvider)`.
- **Artwork** → migrate to `registry.locked(ArtworkProvider, match)`.

Document the inventory in the sub-phase report.

Commit: `docs(indexer): audit backfill_ids.py provider usage for registry migration`

### 11.2 — Migrate ratings aggregation to `fan_out`

For each rating-fetch site, replace direct calls with the fan_out pattern from DESIGN §6.3:

```python
ratings_result = registry.fan_out(RatingProvider)  # FanOutResult[RatingProvider]
collected_notations = []
for provider in ratings_result.values:
    try:
        if not provider.knows(match):
            local_id = registry.cross_ref(match, target=provider.provider_name)
            if local_id is None:
                continue
        else:
            local_id = match.id
        notation = provider.get_notations(local_id, media_type=match.media_type)
        if notation:
            collected_notations.append(notation)
    except (CircuitOpenError, NetworkError):
        continue
```

The aggregated notations replace the per-client `tmdb_client.get_notations()` + `omdb_client.get_notations()` + `imdb_client.get_notations()` + `rt_client.get_notations()` cascade.

Commit: `feat(indexer): migrate backfill_ids ratings aggregation to registry.fan_out`

### 11.3 — Migrate ID validation / cross-ref to registry.get + registry.cross_ref

Direct-dispatch capabilities (IDValidator, IDCrossRef) are correctly accessed via `registry.get(name).validate(id)` and `registry.cross_ref(match, target=name)`. Replace any `tmdb_client.validate_id(...)` or equivalent.

Commit: `feat(indexer): migrate backfill_ids ID validation to registry.get + cross_ref`

### 11.4 — Migrate details / artwork (if applicable)

Same pattern as Phase 7's scraper migration. Use `registry.chain()` / `registry.locked()`.

Commit: `feat(indexer): migrate backfill_ids details + artwork to chain/locked semantics`

### 11.5 — Update `commands/library/scan.py` to pass `registry` instead of individual clients

After 11.2-11.4, `backfill_ids.py` no longer needs typed client parameters. Update `library_backfill_ids` to pass `registry=app_context.provider_registry`:

```python
# Before:
return run_backfill_ids(
    db_path=db_path,
    tmdb_client=tmdb_client,
    tvdb_client=tvdb_client,
    imdb_client=imdb_client,
    rt_client=rt_client,
    ...
)

# After:
return run_backfill_ids(
    db_path=db_path,
    registry=app_context.provider_registry,
    ...
)
```

Remove the 4 `try: ... except UnknownProviderError: ... = None` blocks (they were the symptom of typed-client extraction; no longer needed).

Commit: `feat(indexer): library_backfill_ids passes registry, drops typed-client extraction`
