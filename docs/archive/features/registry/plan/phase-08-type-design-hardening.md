# Phase 8 — Type design hardening

Generated by the post-merge retrospective. Addresses items #4, #5, #6, #7, #8, #28 of the 30-point audit. Type design compromises accepted during cycles 1+2 — collected here for proper resolution.

## Gate

- Phase 7 complete (chain semantics in production — some type system pressure will move once consumers stop using cast()).
- Registry framework stable.

## Goal

Tighten type design to deliver the static guarantees promised by DESIGN §5.2 + §5.3. After this phase, mypy refuses misuse at compile time instead of relying on runtime `WrongSemanticBug`.

## Scope

- `personalscraper/api/metadata/registry/__init__.py` — Mode enum, overload structure, LockedProvider generics, ProviderName unification.
- `personalscraper/api/_contracts.py` — possibly extend the ProviderName Enum (or document the dual-name decision).
- `personalscraper/scraper/confidence.py` — replace `list:  # type: ignore[type-arg]` with proper parameterized types.
- All call sites with `# type: ignore[type-abstract]` / `[attr-defined]` — re-evaluate after the type fixes.

## Sub-phases

### 8.1 — `Mode(str, Enum)` → `StrEnum`

`personalscraper/api/metadata/registry/__init__.py` declares `class Mode(str, Enum)` with a stale "Python 3.10 compat" comment. Project requires `>=3.12` per `pyproject.toml`. Upgrade:

```python
from enum import StrEnum

class Mode(StrEnum):
    CHAIN = "chain"
    FAN_OUT = "fan_out"
    LOCKED = "locked"
    DIRECT = "direct"
```

Verify:

- `str(Mode.CHAIN) == "chain"` (StrEnum) vs `str(Mode.CHAIN) == "Mode.CHAIN"` (Enum-of-str) — likely test assertions adjust.
- `operations()` introspection returns same values.
- `mode_for()` in `_semantics.py` still works.

Commit: `refactor(registry): Mode → StrEnum (Python 3.12+ — drop stale 3.10 compat shim)`

### 8.2 — `@overload` exhaustive partition for `chain()` / `fan_out()` / `locked()`

DESIGN §5.2 promised mypy refusal of `chain(RatingProvider)`. Current overloads use a fallback `type` parameter that defeats the static check. Fix:

```python
# Remove the fallback `def chain(self, capability: type) -> list[Any]:` shape.
# Keep ONLY overloads covering each ChainCapability member:

@overload
def chain(self, capability: type[Searchable]) -> list[Searchable]: ...
@overload
def chain(self, capability: type[MovieDetailsProvider]) -> list[MovieDetailsProvider]: ...
@overload
def chain(self, capability: type[TvDetailsProvider]) -> list[TvDetailsProvider]: ...
@overload
def chain(self, capability: type[EpisodeFetcher]) -> list[EpisodeFetcher]: ...
def chain(self, capability):  # runtime — unannotated
    # ... body unchanged ...
```

After this, `registry.chain(RatingProvider)` produces a mypy error: `No overload variant of "chain" matches argument type "type[RatingProvider]"`. The `WrongSemanticBug` runtime guard becomes belt-and-suspenders.

Same for `fan_out()` (1 overload) and `locked()` (4 overloads — one per LockedCapability member).

Update call sites whose `# type: ignore[type-abstract]` becomes obsolete.

Commit: `refactor(registry): exhaustive @overload partition refuses wrong capability at mypy time`

### 8.3 — `LockedProvider[C]` generic flow preservation

Current `_make_locked` returns `LockedProvider[Any]` because `candidate: object` (from `self._providers.get`). The Generic[C] is lost. Consumer sites use `# type: ignore[attr-defined]`.

Fix: in `locked()`, narrow `candidate` via `isinstance(candidate, capability)` BEFORE calling `_make_locked`. Mypy then flows `capability` (which is `type[C]`) through `_make_locked` typed as:

```python
def _make_locked(*, provider: C, bound_id: str, source_match: ProviderMatch, translated_via: str | None) -> LockedProvider[C]:
    ...
```

In `locked()`:

```python
for candidate_name in self._index.get(capability, []):
    candidate = self._providers.get(candidate_name)
    if candidate is None:
        continue
    if not isinstance(candidate, capability):  # narrows candidate to type capability
        continue
    if not _eligible(candidate):
        continue
    # candidate is now typed as capability (e.g. ArtworkProvider)
    return _make_locked(provider=candidate, ...)  # LockedProvider[ArtworkProvider]
```

Consumer sites lose `# type: ignore[attr-defined]`:

```python
locked = self._registry.locked(ArtworkProvider, match)
urls = locked.provider.get_artwork_urls(locked.bound_id, ...)  # mypy: knows provider is ArtworkProvider
```

Commit: `refactor(registry): preserve Generic[C] flow through _make_locked + locked()`

### 8.4 — Resolve `ProviderName` dual-name decision

Two options — pick one and document:

**Option A — Unify on Enum**: extend `api/_contracts.ProviderName` Enum to include IMDB + ROTTEN_TOMATOES. Drop `RegistryProviderName` NewType. Update all sites. Rewrite the 12 test fixtures that use synthetic names like `"multi"` / `"xref"` to use real Enum values (e.g. extend Enum with TEST_ALPHA/TEST_BETA/...).

**Option B — Document the dual-name**: keep `RegistryProviderName` NewType for the open-string registry world; keep `ProviderName` Enum for the closed-string transport-config world. Add a CLAUDE.md note + a one-paragraph rationale in DESIGN §5.3 explaining why the two coexist.

Recommended: **Option A** for design purity. The Enum extension is small, and Test fakes should be the ones adapting to production types, not the other way around.

Commit: `refactor(registry): unify ProviderName on api/_contracts.py Enum (Option A)` OR `docs(registry): document RegistryProviderName / ProviderName dual-naming (Option B)`

### 8.6 — Document the sentinel-token `LockedProvider` trade-off

The current `LockedProvider._token: object = field(repr=False)` is a public-ish frozen-dataclass field. Visible in `dataclasses.fields()`, in `==`, in `asdict()`. The `__post_init__` guard works at runtime but the field pollution is real.

Two paths:

- (a) Add a docstring note in `LockedProvider` explaining the trade-off (pragmatic, no code change).
- (b) Switch to `__init__`-time sentinel checking via a private `_construct()` classmethod that uses `object.__new__` + `object.__setattr__` (no public `_token` field at all).

Recommended: **(a)** — the trade-off is acceptable and the runtime guarantee holds. The lint risk (downstream code touching `_token`) is low because the underscore prefix is universally honored.

Add to `LockedProvider` docstring:

```
.. note::

    The ``_token`` field is a runtime sentinel — it should never be referenced
    by external code. The underscore prefix is the only convention enforcement;
    a stricter ``__new__``-based hide would clutter the dataclass without
    materially improving safety. Trade-off accepted at design time
    (cycle 1 review, 2026-05-27).
```

Commit: `docs(registry): document LockedProvider sentinel-token trade-off`
