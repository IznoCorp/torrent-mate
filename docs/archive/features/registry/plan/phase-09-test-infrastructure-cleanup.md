### 9.1 â€” Remove `_patch_provider_registry_for_cli_tests` autouse fixture

The autouse fixture in `tests/conftest.py:151-197` patches `ProviderRegistry` globally for CLI E2E tests because `mock_settings.return_value = MagicMock()` propagates into the registry's provider builders (TMDBClient policy + transport) which fail on JSON-serializing MagicMock attributes.

The clean fix is a **typed Settings stub fixture** used everywhere:

```python
# tests/fixtures/settings_stub.py (NEW)
@pytest.fixture
def typed_settings_stub() -> Settings:
    """Minimal valid Settings â€” covers all credential fields the registry needs."""
    return Settings(
        tmdb_api_key="dummy_tmdb_key",
        tvdb_api_key="dummy_tvdb_key",
        omdb_api_key="dummy_omdb_key",
        trakt_client_id="dummy_trakt_id",
        # ... fill all fields with realistic dummy values ...
    )
```

Then sweep CLI test files (~90 files) replacing `mock_settings.return_value = MagicMock()` with `mock_settings.return_value = typed_settings_stub`. After the sweep, the autouse fixture is no longer needed â€” `ProviderRegistry` boots happily on the stub.

This is a 90-file sweep. Recommended dispatcher: Opus 1M (cross-cutting) OR sharded DeepSeek (15-20 files per shard).

Commit: `refactor(tests): typed Settings stub replaces autouse ProviderRegistry patch`

### 9.2 â€” `_eligible` allowlist hardening â€” replace Fake\* prefix heuristic

Current heuristic in `_factory.py:255-257`:

```python
cls_name = type(provider).__name__
if cls_name.startswith("Fake") or cls_name.startswith("_Fake"):
    return True
```

A production class accidentally named `Fake*` would silently bypass. Replace with explicit opt-in marker:

```python
# On any test fake that declares no `.circuit`:
class FakeSearchable:
    provider_name: ClassVar[str] = "fake_searchable"
    _registry_test_fake: ClassVar[bool] = True  # explicit opt-in

# In _factory.py _eligible():
if getattr(provider, "_registry_test_fake", False) is True:
    return True
```

Sweep `tests/` for fake classes lacking `.circuit` â€” add `_registry_test_fake: ClassVar[bool] = True`.

Commit: `refactor(registry): explicit _registry_test_fake marker replaces Fake* prefix heuristic`

### 9.3 â€” Helper class scoping for `_BrokenIDCrossRef` / `_FailingTransportIDCrossRef` / `FakeRatingForTest` etc.

Module-level helper classes in `test_registry_locked.py` and `test_registry_http.py` survive between tests. While pytest isolates state via fixtures, the class objects themselves are shared. Cleaner pattern: move them inside the test function OR into a per-test conftest.

Audit: any cross-test contamination via these helpers? Likely not (no class-level state), but the pattern invites it.

Recommendation: move all `_Fake*` / `_Broken*` helper classes into a single `tests/unit/api/metadata/registry/conftest.py` section labeled `# --- Test helpers --` with comments documenting their per-test purpose.

Commit: `refactor(tests): consolidate registry test helper classes in conftest`

### 9.5 â€” Strengthen designâ†’test mapping
