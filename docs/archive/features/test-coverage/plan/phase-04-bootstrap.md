## Task 4.3 — Write `tests/integration/test_design_api_transport.py`

**Files created**: `tests/integration/test_design_api_transport.py`

```python
"""Design-contract tests for the api-unify transport layer."""

from __future__ import annotations

import pytest

from personalscraper.api._contracts import ApiError, CircuitOpenError
from personalscraper.api.transport._http import HttpTransport
from personalscraper.api.transport._policy import (
    CircuitPolicy,
    RateLimitPolicy,
    RetryPolicy,
    TransportPolicy,
)
from personalscraper.api.transport._auth import NoAuth


class TestCircuitBreakerContract:
    """Design-contract tests for circuit breaker behavior."""

    def test_circuit_opens_after_threshold_failures(self, mock_api_server) -> None:
        """Circuit breaker opens after N consecutive failures.

        Design: docs/features/api-unify/DESIGN.md#circuit-breaker
        Contract: After ``CircuitPolicy.failure_threshold`` consecutive
        failures, the circuit transitions from CLOSED to OPEN. Subsequent
        requests within the cooldown window raise ``CircuitOpenError``
        without reaching the underlying transport.
        """
        threshold = 3
        mock_api_server.queue_responses(
            [(503, '{"error": "Unavailable"}')] * (threshold + 1)
        )

        policy = TransportPolicy(
            provider_name="test",
            base_url=mock_api_server.url,
            auth=NoAuth(),
            timeout_seconds=2.0,
            retry=RetryPolicy(max_attempts=1),
            circuit=CircuitPolicy(failure_threshold=threshold, cooldown_seconds=60.0),
            rate_limit=RateLimitPolicy(requests_per_second=100.0),
        )

        with HttpTransport(policy) as transport:
            for _ in range(threshold):
                with pytest.raises(ApiError):
                    transport.get("/test")

            with pytest.raises(CircuitOpenError):
                transport.get("/test")
```

The exact anchor (`#circuit-breaker` vs `#circuit-breaker-opens-after-3-failures` etc.) is determined by reading the actual heading in `docs/features/api-unify/DESIGN.md` at implementation time. Use `audit_design_coverage.py` output as the source of truth.

3-step onboarding for a dev who wants to add a new contract test:

````markdown
# Adding a design-contract test (3-step quick-start)

## 1. Identify the anchor

Pick the DESIGN.md (or reference doc) section you want to validate. The
"anchor" is the GitHub-style URL fragment for that heading. Get it from:

    python3 scripts/audit_design_coverage.py | grep -i "<keyword>"

## 2. Write the test

Create `tests/integration/test_design_<module>.py` (or append to an existing
one). The test docstring MUST have both markers:

```python
def test_my_contract(staging_dir):
    """Short behavioral description.

    Design: docs/features/<feature>/DESIGN.md#<anchor>
    Contract: One sentence describing the observable behavior the test
    pins down.
    """
    ...
```
````

## 3. Commit

    git add tests/integration/test_design_<module>.py
    git commit -m "test(<scope>): contract test for <behavior>"

The pre-commit hook regenerates `tests/feature_map/<codename>.json` and
includes it in the commit. CI's `design-gaps` job verifies the map is in
sync.

