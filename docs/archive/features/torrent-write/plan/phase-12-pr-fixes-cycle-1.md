### 12.1 — Bencode `info_hash`: structural rewrite + hardening (C1, Md2, Md3, Md4)

**File:** `personalscraper/api/torrent/_base.py` + `tests/unit/test_torrent_source.py`

- **C1**: replace the flat `data.find(b"4:info")` in `_bencode_info_hash` with a
  **structural top-level dict walk**: confirm `data[0:1] == b"d"`, then iterate
  key/value pairs (key = bencoded string; value end via `_bencode_end`); locate
  the value whose key is exactly `b"info"` at the **top level**; hash
  `data[value_start:value_end]`. Never anchor on a substring inside a value.
- **Md2**: in `_bencode_end`, bound-check the string branch
  (`end = col + 1 + length`; raise `ValueError` if `end > len(data)`); add a
  recursion-depth guard (e.g. cap nesting, raise on exceed).
- **Md3**: in `_parse_magnet_hash`, also accept 32-char base32 btih
  (`[A-Za-z2-7]{32}`): `base64.b32decode` → `.hex()` (lowercase). Keep 40-hex path.
- **Md4**: in `TorrentSource.__post_init__`, reject empty-but-present values
  (`magnet=""` / `file_bytes=b""`) — treat empty as "not set" so exactly-one
  still rejects neither/both AND empties.
- **Regression tests** (must fail before fix): (a) torrent with `announce` +
  `comment` (value containing the bytes `4:info`) + `created by` keys before
  `info` → `info_hash == sha1(info)`; (b) `announce-list` (list-of-lists) before
  `info`; (c) base32 magnet → correct lowercase hex; (d) `TorrentSource("")` and
  `from_file(b"")` → `ValueError`; (e) truncated length → `ValueError`. Prefer a
  tiny real `.torrent` golden fixture for (a).

Commit: `fix(torrent-write): structural bencode info_hash walk + base32 magnet + empty-source guard`

  add a corrupt-payload test (`UnsupportedMediaType415Error` → `ApiError`).
- (mn2, optional) align `apply_limits`/controller param naming if trivial.

Commit: `fix(torrent-write): qBit add() inspects result, handles Conflict409 duplicate, raises on failure (D7/D8)`

  `seeding_time_limit` is **minutes**; map `seed_time_minutes` 1:1.
